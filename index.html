Key Insight

MCP creates a standardized bridge between AI intelligence and real-world functionality, enabling seamless integration without custom development for each use case.

Transport Types

Stdio Transport

  • Local communication (same machine)
  • Uses standard input/output streams
  • Full bidirectional communication
  • Best for development and local tools
  • Simple setup and debugging

StreamableHTTP Transport

  • Remote communication over HTTP
  • Enables publicly accessible servers
  • Uses Server-Sent Events (SSE) for bidirectional messaging
  • More complex but enables cloud deployment
  • Supports horizontal scaling

Choosing the Right Transport

Use Stdio for local development and testing. Use StreamableHTTP for production deployments and when you need to share MCP servers across networks.

Message Types

Request/Result Pairs

  • Always come together (request ‚Üí result)
  • Examples: call_tool_request, initialize_request
  • Synchronous communication pattern

Notifications

  • Events that don't need responses
  • Examples: progress updates, logging messages

HTTP Transport Challenges

The Problem:

HTTP is naturally unidirectional (client ‚Üí server), but MCP needs bidirectional communication

The Solution:

  • Server-Sent Events (SSE) for server ‚Üí client messages
  • Session IDs to track connections
  • Two SSE connections: long-lived for notifications, short-lived for responses

Configuration Flags:

  • stateless=true: For horizontal scaling (breaks some features)
  • JSON response=true: Disables streaming (simpler but less interactive)

Important

Use the same transport in development as planned for production to avoid deployment issues.
